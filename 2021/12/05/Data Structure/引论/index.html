类成员的可见性

初始化类成员均为private。private的类成员仅可被该类的方法访问，而public的类成员可以被任何方法访问。

种类 可见性
public 公有
private 仅可在所在类中使用，对象不能使用
protect 可被所在类及其子类使用，对象不能使用

构造函数的执行过程可分为两个阶段：

  1. 初始化阶段
  2. 计算阶段

初始化列表和构造函数内部赋值有什么区别？

  1. 构造函数赋值并不是显式表示
  2. 对于非内置类（非int、float等）应使用初始化列表，更省时
  3. 如果一个数据成员是不具有零参数的构造函数的类类型，那么，该数据成员也必须在初始化列表里进行初始化。

成员函数

成员函数分为访问函数(accessor) 和 **修改函数(mutator)**。

种类 含义
访问函数 只进行检测但不改变其对象的状态的成员函数
修改函数 改变其对象的状态的成员函数

默认情况下，成员函数均为修改函数，要使成员函数变为访问函数，必须在成员函数参数括号后加上const。

接口与实现的分离

然而在C+＋中，将类的接口与其实现分离是很常见的。接口（.h文件）列出了类及其成员（数据和函数），而实现（.cpp，.c等）提供了函数的具体实现。

以下是该过程的注意事项：

  1. 预处理命令
    接口通常都放在以． h结尾的文件中。需要接口信息的源代码必须include接口文件。为了防止重复读取接口，在头文件ifndef``define``endif一下，可避免接口重复读取。

  2. 作用域运算符
    使用ClassName::member以使用某类的成员函数。

    注：当然也可以使用using namespace ClassName将其变为全局的。

  3. 签名必须精确匹配

  4. 如基本类型一样声名对象

vector 和 string

尽量用 vector 和 string， 而不用C++内置数组和内置字符数组。
